# Security Policy

## Responsible Use

This project is intended for:
- Security education
- Authorized penetration testing
- Development auditing

Any misuse is strictly prohibited. The authors and contributors are not responsible for any damage caused by the misuse of this tool.

## Reporting a Vulnerability

If you discover a security vulnerability within this tool itself, please open an issue or contact the maintainers directly. We take all security reports seriously and will investigate them promptly.

## Best Practices

When using this tool, always:
1. Ensure you have explicit permission to test the target system.
2. Use strong, unique secrets for your own applications.
3. Keep your wordlists updated with known weak patterns.
